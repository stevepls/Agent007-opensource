## 2. Chat API → CrewAI Migration

### Problem
The Chat API (`api_chat.py`) was calling Claude/OpenAI directly and executing tools via `ToolRegistry.execute()`. CrewAI was only used for dev tasks (`run_dev_task`). This meant time tracking, task management, communication queries, etc. all bypassed the agent framework.

### Solution
All Chat API requests now route through a new **Orchestrator Crew** (`crews/orchestrator_crew.py`).

### Architecture Change

**Before:**
```
Dashboard → Chat API → Direct LLM (Claude/OpenAI) → Tool Calling → ToolRegistry.execute()
```

**After:**
```
Dashboard → Chat API → CrewAI Orchestrator Crew → Agent with All Tools → ToolRegistry
```

### Implementation Details

- **New file:** `Orchestrator/crews/orchestrator_crew.py`
  - `create_orchestrator_agent()` — Single agent with access to all tools
  - `get_all_crewai_tools()` — Combines existing CrewAI tools (from `tools/` directory) with ToolRegistry tools (dynamically wrapped)
  - `create_tool_wrapper()` — Converts ToolRegistry tools to CrewAI `BaseTool` format
  - `run_orchestrator_task()` — Entry point with governance (audit logging, cost tracking)

- **Modified:** `Orchestrator/api_chat.py`
  - `stream_claude_response()` — Now calls `run_orchestrator_task()` in a thread pool executor (CrewAI is synchronous)
  - `stream_openai_response()` — Delegates to same orchestrator crew
  - Both providers now use the same agent pipeline

### Tool Loading (User Improvement)
The user refactored the tool loading to use a cleaner loop pattern:
```python
tool_loaders = [
    ("harvest", get_harvest_tools),
    ("tickets", get_ticket_tools),

for name, loader in tool_loaders:
    try:
        all_tools.extend(loader())
    except Exception as e:
        print(f"[WARN] Failed to load {name} tools: {e}")
```

```

  ├─ ClickUp → Task backlog, velocity, overdue, team workload
  ├─ Harvest → Time utilization, billable %, project hours
  ├─ Gmail → Client communication health, unread count
  ├─ Slack → Team communication activity
  ├─ Zendesk → Support ticket volume, response times
  ├─ Upwork → Freelancer billing, contract status
  └─ QuickBooks → Revenue, P&L, cash flow (needs re-auth)

  ├─ Team Utilization Rate (hrs logged / capacity)
  ├─ Client Health Score (per client)
  ├─ Revenue Per Hour / Per Client
  ├─ Task Velocity (created vs completed per week)
  ├─ Backlog Debt Score (stale/overdue ratio)
  ├─ Communication Health (response times, unread)
  └─ Cash Flow Status (invoiced vs received)

Report Generators (new)
  ├─ SWOT Analysis (auto-generated from metrics)
  ├─ Business Health Snapshot (dashboard JSON)
  ├─ Weekly Executive Summary
  ├─ Client Profitability Report
  └─ Team Performance Report

Output Formats
  ├─ JSON (Dashboard UI cards + charts)
  ├─ Markdown (reports/docs)
  ├─ Google Sheets (spreadsheet analysis)
  └─ Notion Page (team sharing)
```
